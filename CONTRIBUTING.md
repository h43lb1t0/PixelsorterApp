# Contributing to PixelsorterApp

First off, thank you for considering contributing to PixelsorterApp! It's people like you that make this tool better for everyone.

## Getting Started

1. Fork the repository and create your branch from `master`.
2. Ensure you have the .NET 10 SDK and the .NET MAUI workloads installed.
3. Build the solution to make sure everything works before you make any changes.

## Making Changes

- Please write clear, concise commit messages.
- Test your changes locally to ensure they do not break existing functionality.
- Try to adhere to the existing UI styling (e.g., app-style flat layouts for controls, and distinct visual states for buttons).

### ⚠️ Important Note Regarding Apple Platforms (iOS / MacCatalyst)

**The maintainer of this project does not own any Apple devices.** This means that any code specific to iOS or MacCatalyst cannot be locally tested or verified by the maintainer. 

If your pull request contains changes that affect Apple platforms:
- **You are responsible for ensuring the code works.** Please test it thoroughly on an Apple device or simulator.
- Please provide screenshots, screen recordings, or a detailed breakdown of your testing process for these platforms in your PR description.
- Ensure that your changes do not break other platforms (Windows, Android).

## Submitting a Pull Request

1. Push your branch to your fork.
2. Open a Pull Request against the `master` branch.
3. Provide a clear and detailed description of your changes.
4. If your PR resolves an open issue, link it in the PR description so it can be automatically closed.

Thank you for your contributions!
